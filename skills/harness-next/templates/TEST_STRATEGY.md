# 测试策略

> **阶段定位**：详细设计阶段产出。在实现**之前**定义"怎么证明它是对的"。

---

## 测试金字塔

```
        E2E 测试（少量 · 覆盖核心流程）
       ────────────────────────────────
      集成测试（适量 · 覆盖模块间交互）
     ────────────────────────────────────
    单元测试（大量 · 覆盖核心逻辑 & 工具函数）
```

---

## 测试层级定义

### 单元测试

- **覆盖范围**：核心算法、工具函数、独立的业务逻辑
- **目标覆盖率**：[X]%
- **执行频率**：每次提交（pre-commit hook）
- **运行命令**：`[命令]`
- **文件约定**：`[filename].test.ts` 与源文件同目录

**重点覆盖模块**：
- [ ] [模块 A] 的核心逻辑
- [ ] 数据验证函数

---

### 集成测试

- **覆盖范围**：模块间交互、数据库操作、外部 API 集成
- **执行频率**：每次 PR
- **运行命令**：`[命令]`
- **依赖**：测试数据库（Docker Compose）

**测试场景**：
- [ ] 完整的用户注册流程
- [ ] [核心流程 1]

---

### E2E 测试

- **覆盖范围**：核心用户场景
- **执行频率**：每次部署到 staging
- **工具**：[Playwright / Cypress]

**核心场景**：
- [ ] [最重要的用户流程]

---

## 测试数据策略

| 数据类型 | 方案 | 说明 |
|---------|------|------|
| 正常路径 | 工厂函数 / Fixture | 标准输入 |
| 边界值 | 手写用例 | 空值、最大值、特殊字符 |
| 异常场景 | Mock | 服务超时、权限不足 |
