## 核心文档导航

在开始任何任务之前，先读这些文档：

| 文档 | 内容 | 什么时候读 |
|------|------|-----------|
| `ARCHITECTURE.md` | 系统架构约束、模块边界、禁止事项 | **每次开始任务都读** |
| `docs/API_SPEC.md` | 前后端接口契约 | 实现任何接口时 |
| `docs/DATA_MODEL_PHYSICAL.md` | 数据库表结构 | 数据库相关操作时 |
| `docs/TEST_STRATEGY.md` | 测试要求和规范 | 写或修改测试时 |
| `RULES.md` | 代码规范和提交规范 | 提交代码前 |

## 常见任务清单

### 实现一个新接口
1. 读 `docs/API_SPEC.md` 确认接口定义
2. 读 `docs/DATA_MODEL_PHYSICAL.md` 确认数据结构
3. 按 `ARCHITECTURE.md` 中的模块边界组织代码
4. 写单元测试（参考 `docs/TEST_STRATEGY.md`）
5. 确认响应格式符合 `docs/API_SPEC.md` 的约定

### 修改数据模型
1. 先更新 `docs/DATA_MODEL_PHYSICAL.md`
2. 再修改代码，保持一致
3. 如影响接口返回值，同步更新 `docs/API_SPEC.md`

### 发现设计问题
不要擅自改变接口路径、参数格式或数据结构。先通过注释或 PR 描述说明问题，等人类确认后再修改。

## 重要提示

- 遇到不确定的设计决策，**查文档，不要猜**
