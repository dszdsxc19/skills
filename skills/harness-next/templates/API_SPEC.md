# API 规范

> **阶段定位**：详细设计阶段产出。这是前后端的**正式契约**。
> 参考：`docs/SYSTEM_DESIGN.md`（模块划分）和设计稿（数据需求）

---

## 基本约定

- **Base URL**：`/api/v1`
- **响应格式**：
  ```json
  {
    "code": 200,
    "data": {},
    "message": "success"
  }
  ```
- **鉴权方式**：Bearer Token（`Authorization: Bearer <token>`）
- **版本策略**：URL 路径版本化（`/api/v1/...`）

---

## 通用错误码

| 错误码 | HTTP 状态 | 含义 |
|--------|---------|------|
| 40001 | 400 | 参数错误 |
| 40101 | 401 | 未登录/Token 过期 |
| 40301 | 403 | 权限不足 |
| 40401 | 404 | 资源不存在 |
| 50001 | 500 | 服务内部错误 |

### [模块名称]

#### `POST /api/v1/[resource]`

**描述**：[做什么]

**请求参数**：

```json
{
  "field1": "string",
  "field2": 0
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| field1 | string | 是 | |
| field2 | number | 否 | |

**返回示例**：

```json
{
  "code": 200,
  "data": {
    "id": "uuid"
  },
  "message": "success"
}
```

**错误情况**：

| 错误码 | 触发条件 |
|--------|---------|
| 40001 | 参数校验失败 |
